How each output type adapts the creation area

📷 Photo
Prompt: "What should they do?"
Config: Image count (1-4)
Templates: 14 single photo templates
Refs: Backgrounds, products, outfits
Cost: 1 credit per photo
Action: generateContent()
📸 Carousel
Prompt: "What's the carousel about?"
Config: Format picker, slide count, per-slide editing
Templates: 8 carousel formats
Refs: Per-slide reference attachment
Cost: 1 credit per slide
Action: generateCarousel()
🎥 Video
Prompt: "What happens in the video?"
Config: Duration (5/10s), aspect ratio, source photo (optional)
Sub-modes: From text | From photo | Motion transfer
Refs: Backgrounds, source images
Cost: 3-5 credits
Action: generateVideoFromText/Image/MotionTransfer()
🎤 Talking Head
Input: Script (replaces prompt)
Config: Voice picker, duration (15/30s), background setting
AI assist: "Write me a script about..."
Refs: Background references
Cost: 8-12 credits
Action: generateTalkingHead()

Key principles

1. References are universal. Same panel, same refs, regardless of output type. Click a ref → it attaches. Works for photo, video, carousel, talking head.
2. Type selector, not mode switch. Tabs at the top. Your prompt/refs stay when you switch types. Only the config area changes.
3. Templates are shortcuts, not required. Quick-pick chips that pre-fill the prompt. Totally optional. Power users just type.
4. Inspiration photos inline. "I saw this viral photo" — upload it right in the left panel. It becomes a reference for this generation.
5. One generate button. Bottom-right. Shows the cost. Adapts text per type: "Generate Photo" / "Generate 5 Slides" / "Generate Video" / "Generate Talking Head".
