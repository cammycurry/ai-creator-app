### Library V2
- Full-viewport overlay (like studio)
- Wider responsive grid, polished cards with badges/star/overlay
- Reference usage modes: Exact / Same Scene / Vibe
  - Exact: image passed to Gemini, "match exactly"
  - Same Scene: image passed, "similar setting, creative freedom"
  - Vibe: AI reverse-engineers mood description, no image passed — just the aesthetic
- Drag-drop upload on library page (reads file, opens dialog pre-filled)
- Mode selector dropdown on each attached ref in creation panel

### Generation Quality Upgrade
- **Prompt seed system**: Dense identity paragraph per creator, generated by Gemini Flash from base image during wizard
- Stored as `Creator.promptSeed` — injected into every generation as character anchor
- **Structured prompt format**: CHARACTER → ENVIRONMENT → CAMERA → REALISM sections
- **6 validated realism anchors** (from A/B/C testing): stray hairs, fabric texture, environmental noise, lighting, camera characteristics, visible pores
- **Dropped anchors** that hurt quality: under-eye texture, uneven skin tone, nail detail, "avoid airbrushed"
- **Camera profiles**: selfie front cam, rear cam, mirror selfie — never say "Shot on iPhone"
- **Attractiveness reinforcement** woven into every prompt
- **Updated CONTENT_ENHANCE_PROMPT**: Grok now outputs structured sections
- **Updated buildSampleContentPrompts**: new structured format
- All pipelines upgraded: content, carousel, talking head
- Legacy fallback: creators without promptSeed use old trait injection

## Known Issues (from holistic review)

### Important (should fix soon)
1. **Shared RefCard component not extracted** — Library and Studio browser render cards with different markup. Should extract shared component.
2. **processDownload returns base64 through server action** — 5-10MB JSON responses for large images. Should use API route streaming instead.
3. **Aspect ratio not wired to generation** — UI exists, store has the value, but `generateContent()` doesn't accept or use it.
4. **Content browser stale after generation** — doesn't always refresh when new content is created.
5. **Creator switching in studio** — spec said "no switching inside studio" but we added a dropdown. Decide which approach and document.
6. **Mixed CSS prefixes** — V3 CSS file has both sv2- and sv3- rules since creation panel kept old class names.

### Minor
7. `resizable.tsx` is unused (leftover from panel experiments)
8. Library page CSS imported twice (page.tsx + workspace-shell.tsx)
9. No error boundary around studio
10. Carousel deletion from canvas may hit wrong table (Content vs ContentSet)
