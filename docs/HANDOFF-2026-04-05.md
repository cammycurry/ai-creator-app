## What Needs Work Next

### Priority 1: Viral Content Templates (THE FEATURE)
This is the killer feature the user identified. "Copy this viral shit with your AI creator."

**Concept:** A showcase library where each template is a real piece of content (video, photo, carousel) that users can recreate with their creator. Not saved settings — actual examples with preview + one-click generation.

**User flow:**
1. User browses template library (organized by niche/type/trending)
2. Sees example: "Gym GRWM reel" with preview video of an AI creator doing it
3. Clicks "Make this with my creator"
4. Studio opens pre-configured → generates their version
5. Optional: change background, outfit, etc.

**What this requires:**
- Template data model (preview media, generation config, niche, type, popularity)
- Instagram content scraping → template creation pipeline (admin tool)
- Template browse/discover UI in the app
- One-click generation from template
- Video/image analysis to extract generation parameters
- Admin needs to generate a LOT of example content across niches

**Why it matters:** Solves onboarding ("what can this app do?"), solves ideation ("what should I post?"), and solves the core value prop ("copy viral content with your AI person").

**This needs a full brainstorm → spec → plan → execute cycle.**

## Architecture Decisions

- **3-layer content engine:** Layer 1 = generation engine (now), Layer 2 = AI agent (next), Layer 3 = public API (future). All call same structured functions.
- **Input bar = quick create, Studio = power mode** — Photo/Video generate inline from bar. Carousel/Voice open studio (need format/voice picker).
- **References are account-level** — not per-creator. Shared across all creators.
- **Two reference types only** — BACKGROUND and REFERENCE. Tags do sub-categorization.
- **Public refs auto-save on use** — using a public ref copies it to your library automatically.
- **Viral templates > content presets** — the showcase/template library IS the killer feature, not saved configs.
